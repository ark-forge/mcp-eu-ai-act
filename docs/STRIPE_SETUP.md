# Stripe Setup Guide

Operations guide for configuring Stripe billing for the EU AI Act MCP.

## Required Environment Variables

| Variable | Description | Example |
|----------|-------------|---------|
| `STRIPE_SECRET_KEY` | Stripe API secret key (live or test) | `sk_live_...` or `sk_test_...` |
| `STRIPE_WEBHOOK_SECRET` | Webhook endpoint signing secret (generated by Stripe) | `whsec_...` |
| `STRIPE_PRICE_PRO` | Price ID for the Pro plan (€29/month) | `price_1Xyz...` |
| `STRIPE_PRICE_CERTIFIED` | Price ID for the Certified plan (€99/month) | `price_1Abc...` |

Set these in your environment or in a `.env` file (never commit secrets to git):

```bash
export STRIPE_SECRET_KEY="sk_live_..."
export STRIPE_WEBHOOK_SECRET="whsec_..."
export STRIPE_PRICE_PRO="price_1Xyz..."
export STRIPE_PRICE_CERTIFIED="price_1Abc..."
```

## Creating Products in the Stripe Dashboard

### Step 1: Create the Pro Product

1. Log in to [dashboard.stripe.com](https://dashboard.stripe.com)
2. Go to **Product catalog** → **+ Add product**
3. Set:
   - Name: `ArkForge EU AI Act MCP — Pro`
   - Description: `Unlimited compliance scans + CI/CD API`
4. Under **Pricing**, click **+ Add a price**:
   - Pricing model: **Recurring**
   - Price: `29.00 EUR`
   - Billing period: **Monthly**
5. Click **Save product**
6. Copy the **Price ID** (starts with `price_`) → set as `STRIPE_PRICE_PRO`

### Step 2: Create the Certified Product

1. Go to **Product catalog** → **+ Add product**
2. Set:
   - Name: `ArkForge EU AI Act MCP — Certified`
   - Description: `Unlimited scans + Trust Layer certification included`
3. Under **Pricing**:
   - Pricing model: **Recurring**
   - Price: `99.00 EUR`
   - Billing period: **Monthly**
4. Click **Save product**
5. Copy the **Price ID** → set as `STRIPE_PRICE_CERTIFIED`

## Configuring the Webhook Endpoint

The webhook endpoint is `POST /api/webhook` on your MCP server.

### In the Stripe Dashboard

1. Go to **Developers** → **Webhooks** → **+ Add endpoint**
2. Set:
   - Endpoint URL: `https://your-domain.com/api/webhook`
   - Events to listen for (see next section)
3. Click **Add endpoint**
4. Click **Reveal** next to **Signing secret** → copy `whsec_...` → set as `STRIPE_WEBHOOK_SECRET`

### Webhook Events to Listen For

| Event | Trigger | Action Required |
|-------|---------|-----------------|
| `checkout.session.completed` | User completes payment | Activate API key, set `plan` to `pro` or `certified` in `data/api_keys.json` |
| `customer.subscription.deleted` | Subscription cancelled or payment failed | Deactivate API key, set `plan` back to `free` |

Do not add events not in this list — unhandled events will produce 400 errors and Stripe will retry, causing noise.

## Testing with Stripe CLI

Install the Stripe CLI: [stripe.com/docs/stripe-cli](https://stripe.com/docs/stripe-cli)

```bash
# Log in
stripe login

# Forward webhooks to local server
stripe listen --forward-to localhost:8090/api/webhook

# In a second terminal, trigger test events
stripe trigger checkout.session.completed
stripe trigger customer.subscription.deleted
```

The CLI will output each webhook event and the HTTP response your server returns. A `200` response confirms the event was handled correctly.

When testing, use your `sk_test_...` key and `whsec_test_...` signing secret, not the live credentials.

## Verifying a Payment Went Through

### Method 1: Stripe Dashboard

1. Go to **Payments** → find the payment by email or amount
2. Status should be **Succeeded**
3. Click the payment → check **Metadata** for the customer's API key (if you add it during checkout)

### Method 2: API Key Validation

After a successful payment and webhook processing, the customer's API key should be active and on the `pro` or `certified` plan:

```bash
curl -X POST https://your-domain.com/api/verify-key \
  -H "Content-Type: application/json" \
  -d '{"key": "ak_..."}'
```

Expected response:
```json
{"valid": true, "plan": "pro", "email": "customer@example.com"}
```

### Method 3: Check `data/api_keys.json`

```bash
python3 -c "
import json
data = json.load(open('data/api_keys.json'))
for key, info in data.items():
    if info.get('plan') in ('pro', 'certified') and info.get('active'):
        print(key[:12], info.get('email'), info.get('plan'))
"
```

This lists all active paid API keys. If a key appears here with the correct plan, the payment was processed successfully.
